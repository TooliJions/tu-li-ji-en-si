UI 原型 vs 实现代码 — 完整对比分析报告

逐页面比对 ui-prototype.md 中的 13 个页面线框图与实际实现代码

总览

原型定义 13 个页面，实际实现 18 个页面文件。多出的页面为新增功能页面（emotional-arcs、style-manager、fanfic-init、chapters）。

逐页面对比详情

页面 1：仪表盘 Dashboard — 实现度 85%

已实现

  • 三指标卡片（书籍总数/创作中/总字数）+ 第4个"待处理伏笔"
  • 书籍列表 + 进度条
  • 最近活动记录
  • 质量趋势图（BaselineChart组件）
  • "新建书籍"按钮

差距

  • 原型3指标 → 实现4指标（多了"创作中"和"待处理伏笔"）
  • 原型活动为表格 → 实现为列表样式
  • 质量趋势图原型为散点图 → 实现为BaselineChart折线图

页面 2：书籍详情 BookDetail — 实现度 75%

已实现

  • 章节列表 + 状态标签
  • 操作菜单（编辑/合并/拆分/回滚/删除）
  • PollutionBadge（污染隔离橙色边框+斜纹）
  • TimeDial（时间回溯拨盘）
  • 快速操作区域

差距

  • 缺少"连续写章""系统诊断"按钮
  • 缺少底部"统计"行（已完成X章/草稿X章等）
  • 合并/拆分操作直接执行，无确认弹窗
  • 回滚使用快照列表而非原型中旋转拨盘UI

页面 2b：创建新书 BookCreate — 实现度 95%

已实现

  • Step 1 基本信息（书名/题材/语言/平台）
  • Step 2 创作设置（目标章节/字数/模型配置/提示词版本）
  • 创作简报（支持文件上传+文本编辑）
  • 右侧创建摘要面板
  • 题材选项完整（9个题材+4个平台）

差距

  • 原型为分步流程 → 实现基本一致但布局更紧凑
  • 模型配置只显示3个Agent（Writer/Auditor/Planner），原型有更多

页面 4：创作规划 WritingPlan — 实现度 90%

已实现

  • 左侧步骤导航（6步骤）
  • 章节列表 + 状态标识
  • 章节详细规划表单（目标/人物/事件/伏笔）
  • AI辅助生成规划
  • 底部"保存规划"/"开始创作"按钮

差距

  • 原型为双栏（步骤导航+主内容）→ 实现为三栏布局
  • 缺少"当前卷"选择器
  • 缺少"手动编辑"切换模式

页面 5：正文创作 Writing — 实现度 80%

已实现

  • 快速试写（字数+意图+按钮）
  • 完整创作（意图textarea+两个按钮）
  • 流水线进度实时展示（SSE推送）
  • 记忆透视（MemoryWordcloud词云）
  • 心流模式（全屏暗化#1a1a2e+实体高亮）
  • 日志面板 + 草稿转正 + 上下文漂移检测

差距

  • 原型流水线进度为5步圆点 → 实现为6步网格标签
  • 缺少"质量仪表盘"（AI痕迹/连贯性/节奏等8维度进度条）
  • 记忆透视缺少"3秒后自动淡出"动画
  • 心流模式按钮位置不同（原型在顶栏右侧）

页面 5b：快速试写结果 — 实现度 70%

已实现

  • 试写结果展示（字数/耗时/LLM调用次数）
  • 实体词汇高亮
  • 草稿转正按钮
  • 上下文漂移检测弹窗

差距

  • 缺少"丢弃草稿"和"手动编辑"按钮
  • 漂移检测弹窗缺少"使用旧草稿继续"选项
  • 缺少"上下文版本Token"显示

页面 6：章节阅读 ChapterReader — 实现度 85%

已实现

  • 章节内容阅读 + 编辑模式
  • 心流模式（全屏暗化+实体高亮+ContextPopup）
  • 审计报告面板（三级分类+雷达图）
  • 污染隔离警告横幅
  • TimeDial回滚拨盘
  • 上一章/下一章导航

差距

  • 缺少顶部"上一章/下一章/心流"按钮组（按钮在右侧工具栏）
  • 心流模式下审计面板默认显示（原型应隐藏）
  • 缺少"重新审计"按钮

页面 7：守护进程 DaemonControl — 实现度 75%

已实现

  • 状态总览（状态/下一章/间隔/Token限额）
  • 控制面板（起始/目标章节/间隔时间）
  • 运行日志（级别过滤）
  • 通知推送配置（Webhook+触发事件）
  • Token用量进度条

差距

  • 缺少"智能间隔配置"详细面板（RPM阈值/退避上限）
  • 缺少"每日配额保护"独立面板
  • 缺少底部"统计"行（平均X分钟/章/总耗时）
  • 缺少"连续降级"警告
  • 日志使用MOCK数据而非SSE实时推送

页面 8：伏笔面板 HookPanel — 实现度 80%

已实现

  • 健康概览（5指标卡片）
  • 伏笔列表 + 筛选 + 状态/优先级标签
  • 创建伏笔表单
  • 状态切换下拉菜单
  • 意图设置弹窗（最小/最大章节）
  • 唤醒确认弹窗
  • 时间轴Tab（复用HookTimelineWorkspace）

差距

  • 缺少"伏笔健康度"进度条（活跃伏笔/休眠/逾期/回收率）
  • 惊群调度详情折叠面板未实现
  • 缺少"查看唤醒调度日志"按钮
  • 时间轴双轨视图在独立页面（hook-timeline.tsx）

页面 9：数据分析 Analytics — 实现度 90%

已实现

  • 字数统计图表
  • 审计通过率
  • Token用量（按渠道+按章节）
  • AI痕迹趋势
  • 质量基线 + 漂移检测
  • 情感弧线概览
  • 灵感洗牌（3方案卡片）
  • 质量趋势详情（TrendDetailChart）

差距

  • AI痕迹图原型为折线图+琥珀渐变关注区 → 实现为柱状图
  • 缺少"建议气泡"浮动动画
  • 灵感洗牌缺少"采用此方案/微调"按钮
  • 质量基线缺少"重新建立基线"按钮

页面 10：配置 ConfigView — 未找到独立页面

已实现

  • 配置通过路由指向（/config），但代码文件不存在
  • 模型配置分散在BookCreate中

差距

  • config-view.tsx 文件不存在（sidebar.tsx中有路由但无对应页面）
  • 缺少全局Provider配置/API Key管理
  • 缺少Agent模型路由表格
  • 缺少备用Provider状态表格
  • 缺少通知配置（Telegram Bot等）

页面 11：真相文件 TruthFiles — 实现度 70%

已实现

  • 多Tab导航（概览/源码编辑/角色矩阵/副线管理/冲突检查）
  • 投影状态同步状态
  • Markdown导入工具
  • JSON源码编辑器
  • 角色矩阵卡片
  • 伏笔/副线表格
  • 冲突检查与差异详情

差距

  • 原型标签为[当前状态/伏笔/章节摘要/角色矩阵/情感弧线/副线] → 实现不同
  • 缺少"当前世界状态"角色卡片展示（位置/健康/情感/资源）
  • 缺少"从JSON查看"和"回滚到上一章状态"按钮
  • 角色矩阵只显示名称/确信度，缺少详细信息

页面 12b：提示词版本 PromptVersion — 实现度 90%

已实现

  • 当前配置展示 + 版本切换
  • 可用版本表格（版本/日期/变更说明/Agent数）
  • 版本对比工具（选择两个版本+diff展示）
  • 差异高亮显示

差距

  • Diff对比使用模拟数据而非真实API结果
  • 缺少详细的逐Agent提示词变更展示

页面 13：系统诊断 DoctorView — 实现度 85%

已实现

  • 问题清单（ severity标签+修复按钮）
  • Provider健康状态
  • 质量基线状态
  • 重组恢复功能
  • 状态差异对比
  • 刷新诊断按钮

差距

  • 缺少环境检查区块（Node.js版本/磁盘空间/AI连接）
  • 缺少"修复所有"和"仅清理僵尸锁"按钮
  • 缺少按书分组的详细诊断视图
  • 缺少自然语言对比视图弹窗（设定脱节检测）

总结与优先级建议

!

关键缺失（P0）

1. ConfigView 页面完全缺失 — 侧边栏有路由但无对应组件
2. 守护进程日志为MOCK数据 — 应使用SSE实时推送
3. 合并/拆分操作无确认弹窗 — 存在误操作风险

2

重要差距（P1）

1. 写作页面缺少质量仪表盘8维度进度条
2. 数据分析AI痕迹图缺少琥珀渐变关注区和建议气泡
3. 伏笔面板缺少健康度进度条和惊群折叠面板
4. 真相文件缺少"当前世界状态"展示

3

视觉/UX优化（P2）

1. 记忆透视缺少3秒淡出动画
2. 仪表盘活动表格→列表样式差异
3. 灵感洗牌缺少方案操作按钮
4. DoctorView缺少按书分组诊断
